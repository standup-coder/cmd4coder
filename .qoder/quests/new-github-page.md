# GitHub Pages 设计文档

## 项目概述

为 cmd4coder 项目创建一个专业、美观、简约的 GitHub Pages 官网，采用极简主义设计风格，黑白配色，简洁布局，提供项目介绍、功能展示、文档导航和下载入口等基础功能。

## 设计目标

### 核心目标

- 专业性：展示项目的技术实力和完整性
- 美观性：极简主义设计，黑白配色，视觉舒适
- 简约性：信息清晰，布局简洁，无冗余元素
- 可用性：在 GitHub Pages 配置后能够成功运行
- 响应式：支持桌面端和移动端访问

### 非功能性目标

- 加载速度快（无外部依赖，纯静态资源）
- SEO 友好（合理的语义化标签和 meta 信息）
- 易维护性（纯 HTML/CSS/JS，无构建流程）
- 跨浏览器兼容（支持主流现代浏览器）

## 技术方案

### 技术选型

| 技术项 | 选型 | 理由 |
|--------|------|------|
| 站点类型 | 纯静态站点 | 无需构建流程，易于维护和部署 |
| 核心技术 | HTML5 + CSS3 + Vanilla JavaScript | 零依赖，加载快速，完全控制 |
| 布局方案 | CSS Flexbox + Grid | 现代布局方案，响应式友好 |
| 字体方案 | 系统字体栈 | 避免外部字体加载，提升性能 |
| 图标方案 | SVG 内联 | 矢量图形，可定制，无额外请求 |
| 代码高亮 | Prism.js（轻量版） | 轻量级，支持按需加载语言 |
| 部署方式 | GitHub Pages | 官方支持，自动部署 |

## 页面设计

### 整体布局

采用单页面应用（SPA）结构，通过锚点导航实现内容切换，页面结构如下：

```
┌─────────────────────────────────────────────┐

│  Logo    导航链接（特性|快速开始|文档|下载）  │
├─────────────────────────────────────────────┤
│                                             │
│              Hero Section                   │
│            项目名称 + 核心标语               │
│              快速操作按钮                    │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│            Features Section                 │
│          核心特性展示（网格布局）             │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│          Quick Start Section                │
│         安装步骤 + 使用示例代码               │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│          Documentation Section              │
│          文档链接 + 资源导航                 │
│                                             │

│                                             │

│                                             │
├─────────────────────────────────────────────┤
│              Footer（页脚）                  │
│       GitHub 链接 | 许可证 | 联系方式        │
└─────────────────────────────────────────────┘
```

### 页面章节详细设计

#### Header（页头）

**设计要点**：
- 固定在页面顶部，滚动时保持可见
- 左侧：项目 Logo 和名称
- 右侧：导航链接（平滑滚动到对应章节）
- 简约设计：纯黑白配色，细线分隔
- 响应式：移动端转换为汉堡菜单

**内容元素**：
- Logo：cmd4coder 文字 Logo 或简化图标
- 导航项：特性、快速开始、文档、下载、GitHub

#### Hero Section（英雄区）

**设计要点**：
- 全屏或半屏高度，视觉冲击力
- 中心对齐，大标题 + 副标题
- 简洁的介绍文字
- 两个主要操作按钮（CTA）

**内容元素**：
- 主标题：cmd4coder
- 副标题：命令行工具大全 - 面向运维工程师和开发者
- 描述：一句话介绍项目价值（20+ 分类，220+ 精选命令，双模式交互）
- 主按钮：立即下载
- 次按钮：查看文档（链接到 README.md）
- 版本标识：当前版本号 v1.0.0

#### Features Section（特性展示）

**设计要点**：
- 网格布局展示核心特性（2x4 或 3x3）
- 每个特性卡片包含：图标 + 标题 + 简短描述
- 极简图标设计（SVG 线条图标）
- 卡片悬停效果：边框高亮

**内容元素**（基于 README.md 核心特性）：
1. 全面的命令清单：20+ 分类，220+ 精选命令
2. 强大的搜索功能：4 级优先级匹配算法
3. 快速查询：LRU 缓存优化，毫秒级响应
4. 详细的命令说明：用法、选项、示例、风险提示
5. 双模式交互：CLI 和 TUI 两种使用方式
6. 配置管理：收藏、历史记录、自定义配置
7. 导出功能：Markdown 和 JSON 格式导出
8. 跨平台支持：Linux、macOS、Windows

#### Quick Start Section（快速开始）

**设计要点**：
- 左右分栏或垂直布局
- 左侧：安装步骤（数字编号）
- 右侧：代码示例（带语法高亮）
- 代码块：黑底白字，简约终端风格
- 复制按钮：一键复制代码

**内容元素**：
- 安装方式一：下载预编译版本（推荐）
  - 说明：从 Releases 页面下载对应平台可执行文件
  - 平台列表：Linux、macOS、Windows
- 安装方式二：从源码构建
  - 代码示例：git clone + go build 命令
- 基本使用示例
  - CLI 模式示例：列出分类、查看命令、搜索
  - TUI 模式示例：无参数启动进入交互界面

#### Documentation Section（文档导航）

**设计要点**：
- 卡片式布局，展示主要文档链接
- 每个卡片：图标 + 文档标题 + 简短说明
- 链接到 GitHub 仓库中的对应文档

**内容元素**：
- README：项目介绍和使用指南
- ARCHITECTURE：系统架构文档
- CONTRIBUTING：贡献指南
- CODE_OF_CONDUCT：行为准则
- 完整文档：链接到 GitHub 仓库

#### Download Section（下载区）

**设计要点**：
- 突出显示最新版本号
- 多平台下载按钮（网格或列表）
- 每个平台：图标 + 平台名称 + 架构 + 文件大小
- 所有版本链接：指向 GitHub Releases

**内容元素**：
- 版本信息：v1.0.0（从 README.md 获取）
- 平台列表：
  - Linux (amd64)
  - macOS (amd64)
  - macOS (arm64)
  - Windows (amd64)
- 下载链接：指向 GitHub Releases 页面
- 安装说明：简要的安装步骤提示

#### Footer（页脚）

**设计要点**：
- 简洁分隔线
- 中心对齐或左右分栏
- 链接和版权信息

**内容元素**：
- 社交链接：GitHub 仓库、Issue 反馈
- 许可证：MIT License
- 版权信息：© 2024 cmd4coder
- 额外链接：返回顶部

## 设计风格规范

### 色彩方案

采用极简主义黑白配色，辅以灰度过渡：

| 用途 | 颜色值 | 说明 |
|------|--------|------|
| 主文字 | #000000 | 纯黑 |
| 次要文字 | #666666 | 中灰 |
| 辅助文字 | #999999 | 浅灰 |
| 背景 | #FFFFFF | 纯白 |
| 次要背景 | #F5F5F5 | 极浅灰 |
| 边框 | #E0E0E0 | 浅灰边框 |
| 分隔线 | #CCCCCC | 中灰分隔 |
| 强调色 | #000000 | 黑色（用于按钮和链接） |
| 悬停状态 | #333333 | 深灰 |
| 代码背景 | #1E1E1E | 深色代码块背景 |
| 代码文字 | #D4D4D4 | 浅色代码文字 |

### 字体方案

使用系统字体栈，确保跨平台一致性和加载速度：

| 用途 | 字体栈 | 大小 |
|------|--------|------|
| 主标题 | -apple-system, BlinkMacSystemFont, "Segoe UI", "Microsoft YaHei", sans-serif | 48px / 3rem |
| 副标题 | 同上 | 32px / 2rem |
| 章节标题 | 同上 | 24px / 1.5rem |
| 正文 | 同上 | 16px / 1rem |
| 小字 | 同上 | 14px / 0.875rem |
| 代码 | "SF Mono", "Consolas", "Courier New", monospace | 14px / 0.875rem |

### 间距规范

采用 8px 基础网格系统：

| 用途 | 间距值 |
|------|--------|
| 最小间距 | 8px |
| 小间距 | 16px |
| 中间距 | 24px |
| 大间距 | 32px |
| 超大间距 | 48px |
| 章节间距 | 64px |
| 容器最大宽度 | 1200px |
| 内容内边距 | 24px（桌面）/ 16px（移动） |

### 组件样式

#### 按钮

- 主按钮：黑底白字，hover 时背景变深灰
- 次按钮：白底黑字黑边框，hover 时背景变浅灰
- 圆角：2px（轻微圆角）
- 内边距：12px 24px
- 字体粗细：500（Medium）

#### 卡片

- 背景：纯白
- 边框：1px 实线浅灰
- 圆角：4px
- 阴影：无（极简风格）
- 悬停效果：边框变黑

#### 代码块

- 背景：深色（#1E1E1E）
- 文字：浅色（#D4D4D4）
- 边框：无
- 圆角：4px
- 内边距：16px
- 字体：等宽字体
- 包含复制按钮（右上角）

#### 链接

- 默认：黑色，无下划线
- 悬停：下划线
- 访问后：保持黑色

## 交互设计

### 导航交互

- 点击导航链接：平滑滚动到对应章节（使用 CSS scroll-behavior 或 JS 实现）
- 滚动监听：当前查看章节对应的导航项高亮显示
- 移动端：汉堡菜单展开/收起动画

### 代码块交互

- 复制按钮：点击后复制代码到剪贴板
- 复制反馈：按钮文字变化（复制 → 已复制），2 秒后恢复
- 代码高亮：使用 Prism.js 进行语法高亮

### 下载按钮交互

- 悬停效果：背景色变化
- 点击：跳转到 GitHub Releases 页面
- 平台识别：可选功能，根据用户操作系统高亮推荐的下载选项

### 响应式交互

- 断点设置：
  - 移动端：< 768px
  - 平板：768px - 1024px
  - 桌面：> 1024px
- 移动端调整：
  - 导航转为汉堡菜单
  - 网格布局改为单列
  - 字体大小适当缩小
  - 间距适当减小

## 内容策略

### 文案原则

- 简洁明了：每句话传达单一核心信息
- 技术准确：使用准确的技术术语
- 行动导向：引导用户下一步操作
- 避免冗余：去除不必要的修饰词

### 信息层级

1. 核心信息（首屏可见）：
   - 项目名称和定位
   - 核心价值主张
   - 主要操作入口

2. 重要信息（一屏滚动可见）：
   - 核心特性列表
   - 快速开始指南

3. 补充信息（需滚动查看）：
   - 详细文档链接
   - 下载选项
   - 联系方式

### 代码示例选择

基于 README.md 中的示例，选择最具代表性的命令：

- CLI 模式示例：
  ```bash
  # 查看帮助
  go run ./cmd/cli --help
  
  # 列出所有命令
  go run ./cmd/cli list -d ./data
  
  # 搜索命令
  go run ./cmd/cli search file -d ./data
  ```

- TUI 模式示例：
  ```bash
  # 启动交互式界面
  go run ./cmd/cli -d ./data
  ```

- 安装示例：
  ```bash
  # 从源码构建
  git clone https://github.com/cmd4coder/cmd4coder.git
  cd cmd4coder
  go build -o bin/cmd4coder ./cmd/cli
  ```

- `<header>`: 页头
- `<nav>`: 导航
- `<main>`: 主内容

## 性能优化

### 加载性能

- HTML 内联关键 CSS（首屏样式）
- 非关键 CSS 异步加载
- JavaScript 延迟加载（defer 属性）
- 图片优化：SVG 内联或优化压缩
- 避免外部依赖：无 CDN 引用，所有资源本地化

### 运行性能

- CSS 动画使用 transform 和 opacity（GPU 加速）
- 避免强制同步布局
- 事件委托减少监听器数量
- 图片懒加载（可选，如有大量图片）

### 资源大小控制

| 资源类型 | 大小目标 |
|----------|----------|
| HTML | < 20KB |
| CSS（总计） | < 30KB |
| JavaScript | < 50KB（含代码高亮库） |
| 图片/图标 | < 20KB（SVG 优化） |
| 总页面大小 | < 150KB |

## 部署配置

### GitHub Pages 配置

在 GitHub 仓库设置中启用 GitHub Pages：

- Source：选择分支（main 或 gh-pages）和目录（/docs 或 /root）
- 自定义域名：可选配置 CNAME 文件
- HTTPS：启用强制 HTTPS

### 目录结构选择

推荐方案：使用 `/docs` 目录作为 GitHub Pages 根目录

理由：
- 与源代码分离，结构清晰
- 无需单独分支，维护简单
- 符合 GitHub Pages 官方推荐

### 构建和发布流程

由于采用纯静态站点，无需构建流程：

1. 在本地创建 `docs/` 目录
2. 编写 HTML、CSS、JS 文件
3. 提交到 main 分支
4. 在 GitHub 仓库设置中启用 GitHub Pages
5. 选择 main 分支的 /docs 目录
6. 等待自动部署（通常 1-2 分钟）
7. 访问 `https://[username].github.io/cmd4coder/` 验证

### 域名配置（可选）

如需自定义域名：

1. 在域名服务商配置 CNAME 记录指向 `[username].github.io`
2. 在 `docs/` 目录创建 CNAME 文件
3. 文件内容：自定义域名（如 `cmd4coder.dev`）
4. 在 GitHub Pages 设置中填写自定义域名
5. 等待 DNS 生效和 SSL 证书颁发

## 可访问性

### 基础要求

- 语义化 HTML 结构
- 合理的标题层级（h1 - h6）
- 图片包含 alt 属性
- 链接包含有意义的文字（避免"点击这里"）
- 表单元素包含 label
- 足够的色彩对比度（黑白配色天然满足）

### 键盘导航

- 所有交互元素可通过 Tab 键访问
- 焦点状态清晰可见
- 逻辑顺序的 Tab 导航
- 汉堡菜单可通过键盘操作

### ARIA 属性

在必要位置添加 ARIA 属性：

- `aria-label`: 为图标按钮提供文字说明
- `aria-expanded`: 汉堡菜单展开状态
- `aria-current`: 当前激活的导航项
- `role`: 必要时补充语义角色

## 浏览器兼容性

### 目标浏览器

支持以下现代浏览器的最新两个主要版本：

- Chrome / Edge（Chromium）
- Firefox
- Safari
- 移动端：iOS Safari、Chrome Android

### 降级策略

- 使用 CSS 特性检测（@supports）
- 核心功能在不支持 JavaScript 时仍可访问
- 渐进增强：基础体验 → 增强体验

### 不支持的浏览器

- Internet Explorer（已停止支持）
- 老旧版本移动浏览器（> 3 年）

## 维护和更新

### 内容更新流程

当项目信息变化时（如版本更新、新增特性）：

1. 更新 HTML 文件中的对应内容
2. 更新版本号
3. 更新下载链接（指向新的 Release）
4. 提交变更到仓库
5. GitHub Pages 自动重新部署

### 需要同步更新的内容

- 版本号：与 README.md 保持一致
- 特性列表：与 README.md 核心特性同步
- 下载链接：指向最新 Release
- 命令示例：与文档保持一致

### 更新检查清单

| 检查项 | 位置 | 更新频率 |
|--------|------|----------|
| 版本号 | Hero Section、Download Section | 每次发布 |
| 下载链接 | Download Section | 每次发布 |

## 风险和限制

### 技术限制

- 纯静态站点，无后端功能（如用户登录、数据统计）
- 无法实现实时内容更新（需要重新部署）
- GitHub Pages 流量限制（每月 100GB，通常足够）
- 不支持服务端渲染（SSR）

### 内容限制

- 仅展示基础信息，详细文档需跳转到 GitHub
- 无在线 Demo 或交互式演示
- 无搜索功能（站内搜索）

### 维护考虑

- 需手动更新内容（无自动化同步）
- 版本信息、下载链接需与 GitHub Release 保持同步
- 设计更新需要前端开发知识

### 缓解措施

- 内容更新检查清单：确保同步更新
- 定期检查链接有效性：避免死链
- 保持简单：降低维护复杂度
- 文档化：记录更新流程和注意事项
